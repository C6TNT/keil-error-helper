# keil-error-helper

一个面向 `Keil / C51 / 蓝桥杯单片机模板工程` 场景的轻量 Windows 诊断工具。

它的目标不是自动改代码，而是帮助初学者更快完成这些事：
- 从一长段编译输出里提取第一条真正错误
- 判断错误属于哪一类
- 给出简单直接的中文解释
- 告诉用户先检查哪几个位置
- 结合蓝桥杯模板工程判断更像是哪一层的问题
- 提醒这类错误最容易把模板哪里改坏
- 一键生成适合发给学长或群里的求助文本
- 在需要时调用 AI 做更深入的解释和排查建议

## 运行方式

### 1. 安装依赖

```powershell
pip install -r requirements.txt
```

### 2. 启动桌面版

```powershell
python app/main.py
```

或者直接双击：

- `run_app.bat`

### 3. 命令行测试

```powershell
python app/cli.py
```

然后直接粘贴 `Keil` 报错输出。

## 当前支持的能力

- 提取第一条关键 `error`
- 提取出错文件和行号
- 识别常见 `Keil/C51` 错误类型
- 根据文件路径定位更像是 `App / BSP / Devices / Drivers / Examples` 哪一层的问题
- 结合模板常见坑点给出额外提醒
- 生成适合发给学长或群里的结构化求助文本
- 内置示例报错，第一次打开也能直接体验
- 支持按“页面 / 按键 / 参数 / 温度 / 频率 / 超声波 / 显示”给出更贴场景的建议
- 提供卡片化结果展示和优先级提示
- 提供 AI 深入分析

输出内容包括：
- 错误原文
- 错误类型
- 中文解释
- 最可能原因
- 建议先检查的位置
- 模板定位建议
- 模板常见坑点提醒
- 下一步动作
- 优先级提示

## 使用建议

- 优先粘贴完整 Build Output
- 先只修第一条错误
- 修完后重新编译，再看新的第一条错误
- 如果准备求助，优先用“复制求助文本”按钮
- 如果是第一次打开，可以先载入内置示例体验一遍
- 如果想用 AI，先点“AI 设置”填好 Key、Base URL 和 Model

## AI 使用方式

从 `V0.5` 开始，推荐直接在应用内配置：

1. 打开 App
2. 点击 `AI 设置`
3. 填入：
   - API Key
   - Base URL
   - Model
4. 保存后直接使用 `AI 深入分析`

配置会保存在应用目录下的 `config.json` 里。  
对新生来说，不再需要打开终端配置环境变量。

当前策略是：

1. 先走本地规则分析
2. 用户主动点击“AI 深入分析”
3. 再把结构化输入发给模型做增强解释
