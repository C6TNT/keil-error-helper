# keil-error-helper

一个面向 `Keil / C51 / 蓝桥杯单片机模板工程` 场景的轻量 Windows 诊断工具。

它的目标不是自动改代码，而是帮助初学者更快完成这些事：
- 从一长段编译输出里提取第一条真正错误
- 判断错误属于哪一类
- 给出简单直接的中文解释
- 告诉用户先检查哪几个位置
- 结合蓝桥杯模板工程判断更像是哪一层的问题
- 提醒这类错误最容易把模板哪里改坏
- 一键生成适合发给学长或群里的求助文本
- 在需要时调用 AI 做更深入的解释和排查建议

## 当前支持的能力

- 提取第一条关键 `error`
- 提取出错文件和行号
- 识别常见 `Keil/C51` 错误类型
- 根据文件路径定位更像是 `App / BSP / Devices / Drivers / Examples` 哪一层的问题
- 结合模板常见坑点给出额外提醒
- 生成适合发给学长或群里的结构化求助文本

- 支持按“页面 / 按键 / 参数 / 温度 / 频率 / 超声波 / 显示”给出更贴场景的建议

- 提供 AI 深入分析
- 提供 AI 设置和连接测试
- 支持补充报错附近代码片段，让 AI 分析更贴近真实代码上下文
- 兼容源码运行与 `PyInstaller` 单文件 exe 运行
- 当分析阶段出现异常时，直接提示异常信息而不是静默无响应

## 使用建议

- 优先粘贴完整 Build Output
- 先只修第一条错误
- 修完后重新编译，再看新的第一条错误
- 如果准备求助，优先用“复制求助文本”按钮
- 如果是第一次打开，可以先载入内置示例体验一遍
- 如果想用 AI，先点“AI 设置”填好 Key、Base URL 和 Model
- 填完后先点一次“测试连接”，确认这组配置真的能用
- 如果愿意，再把报错附近几行代码补进“补充代码片段（可选）”，AI 深入分析会更准确

## AI 使用方式

从 `V0.5` 开始，推荐直接在应用内配置：

1. 打开 App
2. 点击 `AI 设置`
3. 填入：
   - API Key
   - Base URL
   - Model
4. 保存后先点 `测试连接`
5. 测试通过后再使用 `AI 深入分析`

配置会保存在应用目录下的 `config.json` 里。  
对新生来说，不再需要打开终端配置环境变量。

当前默认配置已经调整为更适合实际试用的魔搭兼容方案：

- `Base URL`：`https://api-inference.modelscope.cn/v1`
- `Model`：`Qwen/Qwen3.5-35B-A3B`

当前策略是：

1. 先走本地规则分析
2. 用户主动点击“AI 深入分析”
3. 再把结构化输入发给模型做增强解释
